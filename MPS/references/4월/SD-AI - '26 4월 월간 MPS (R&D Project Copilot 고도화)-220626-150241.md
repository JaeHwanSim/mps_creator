# AI - '26 4월 월간 MPS (센터 MPS — R&D Project Copilot 고도화)

> **2026년 4월 월간 MPS (R&D 센터 AI)**
> **과제:** R&D Project Copilot 고도화 (실사용 기준선 v1.0 확보 및 팀 단위 운영 정착)
>
> *센터 MPS(입력 자료). 내 MPS(`MPS/created/`) 도출 시 참고.*

## Mission

3월에 확립한 Gate·Agent 운용 기반 위에서, 4월은 R&D Project Copilot 고도화에 집중한다.
Copilot을 프로젝트 운영의 실질 도구로 만들어, 일정/리스크 예측과 작업 동기화를 자동화하고 팀 단위 일상 운영에 정착시킨다.

## 목적

1. 프로젝트 진행률·리스크 자동 감지와 예측 정확도 향상
2. JIRA·Repo·CI와의 양방향 동기화로 실시간 추적성 강화
3. 팀 주간 운영 루틴에 Copilot을 내재화하여 의사결정 리드타임 단축

## 수요자 요구 사항

- 지연 징후를 사전에 포착하고 대응할 수 있는 알림·대시보드 제공
- 회의·보고를 줄이고도 상태 공유가 가능한 신뢰 가능한 단일 화면 확보
- 프로젝트 문헌·이슈·코드 연결을 통한 근거 기반 의사결정

## 현재 상태

### As-Is
- Copilot 핵심 모듈(Tracker·Sync·Alert)이 개별 동작하나, 팀 단위 일상 운영에 완전히 내재화되지 않음
- 지연 예측 신호가 산발적이며, 거짓 경보와 누락이 혼재
- 운영 대시보드와 PR/이슈 흐름의 연결성이 불충분

### To-Be
- Copilot v1.0이 팀 주간 운영의 표준 도구로 사용되고, 대시보드 한 화면으로 상태·리스크·액션이 정합
- 지연 예측 신뢰도 향상(거짓 경보 감소)과 3일 전 사전 알림 정착
- 이슈–커밋–PR–테스트–릴리즈가 추적 ID로 자동 연결

## 기대하는 결과물

1. **R&D Project Copilot v1.0** (Tracker·Sync·Alert·Dashboard 통합)
2. **운영 플레이북 v1.0** (역할·루틴·품질규칙·알림 규칙)
3. **Pilot 운영 리포트 v1.0** (2개 팀 적용, 예측 정확도·리드타임·이슈 처리율 개선 지표)

## Performance Objectives

### 성과 목표 1: R&D Project Copilot v1.0 출시

> Tracker·Sync·Alert·Dashboard를 통합해 팀 운영 기준선(v1.0)을 완성한다.

**고정변수목표**
- 모듈 통합 빌드 v1.0: Tracker(진척/지연 신호), Sync(JIRA·Repo·CI 양방향), Alert(슬랙·메일), Dashboard(상태/리스크/액션)
- 데이터 스키마 및 추적 규칙 v1.0: 이슈키–브랜치–커밋–PR–테스트–릴리즈 자동 연결
- 알림 정책 v1.0: 임계치·쿨다운·소유자 라우팅·소음 차단 규칙

**공략전략 (고정)**
- 표준 PR 템플릿·브랜치 규칙에 추적 ID 필수화
- 대시보드 카드: 진행률, 지연 Top N, 리스크 신호, 미해결 액션, 품질게이트 상태
- QA 기준: 이벤트 누락률 ≤ 2%, 알림 중복률 ≤ 10%

**변동변수목표**
- 지연 예측 신뢰도 향상: 경보 정밀도 70% 이상, 재현율 60% 이상
- 지표 개선: PR 리뷰 리드타임 20% 단축, 해결까지의 평균 시간(MTTR) 15% 단축

**공략전략 (변동)**
- 실데이터 기반 임계치 재보정(프로젝트별 베이스라인 학습)
- 거짓 경보 원인 분류(휴면 이슈, 계획적 홀드, 데이터 결손) 후 규칙 보강
- 주간 회고로 모델/룰 파라미터 신속 튜닝

**Risk**
- **내부(품질):** 거짓 경보 과다로 현장 신뢰 하락 → 쿨다운·집계 임계치 적용, 프로젝트별 기준선 분리, 회고 기반 재보정
- **내부(데이터):** 이슈–코드 추적 ID 누락 → PR 템플릿 강제·미기재 시 CI 실패, 자동 보정 훅 제공

### 성과 목표 2: 팀 단위 운영 정착 (운영 플레이북 v1.0 + Pilot 리포트)

> 2개 팀 파일럿으로 일상 운영에 Copilot을 내재화하고, 개선 루프를 고정한다.

**고정변수목표**
- 운영 플레이북 v1.0: 역할(R&R), 주간 운영 루틴, Exception 핸들링, 품질게이트
- Pilot 적용 확정: 2개 팀, 최소 1개 스프린트 동안 전 흐름(이슈→PR→테스트→릴리즈) 적용
- 운영 대시보드 v1.0: 팀별 실시간 화면 배포 및 홈고정

**공략전략 (고정)**
- 주간 운영 회의에서 Copilot 대시보드 기반 의사결정만 허용
- 액션 아이템은 Copilot에서 직접 생성·할당·만료 추적
- 교육 세션 1회 + 질문응답 Office Hour 주 1회 운영

**변동변수목표**
- Pilot 운영 리포트 v1.0: 예측 정밀/재현, 리드타임, 누락/중복 알림, 사용자 피드백
- 사용 활성화: 파일럿 기간 주당 대시보드 활성 사용자 비율 80% 이상

**공략전략 (변동)**
- 핵심 사용자 2명/팀 지정하여 현장 피드백 수집·즉시 반영
- 거부감 요인(알림 피로·입력 부담) 최소화를 위한 필드 축소·자동수집 확대

**Risk**
- **외부:** 우선순위 변경으로 파일럿 일정 단축 → 범위 최소화(핵심 흐름만), 주간 의사결정으로 스코프 조정
- **내부(운영):** 도구 분산으로 대시보드 정합성 의심 → 단일 데이터 소스 원칙, 불일치 자동 감지 카드 제공
- **내부(보안):** 프로젝트 메타데이터 노출 우려 → 권한 기반 가시성, 민감 필드 마스킹, 접근 로깅
