# Device - '26 3월 월간 MPS

> **2026년 3월 MPS Planning (Wearable Navigation) — EA(Engineering Approval) Month / SIDEX 데모 구체화**

## Mission

5월 SIDEX 데모 시스템을 Gate 기반 개발 프로세스(EA 단계)에 맞춰 구체화하여,

a. Demo System 기구 디자인과 S/W 디자인을 완료하고
b. 개발 Agent를 활용한 Main Navigation 화면 S/W 구현 결과물을 확보한다.

> **참고:** Gate 기반 프로세스에서 EA 단계의 핵심 산출물은 SRS, Architecture, SDS 이며, Design Input 완결성 검증과 아키텍처 확정이 목표이다.

## 개요

2월 PA에서 합의된 Scope/마일스톤을 기준으로 3월은 EA(설계 승인) 단계를 수행하는 달이다.
5월 SIDEX 전시를 위해 “무엇을 만들지”를 SRS로 고정하고, “어떻게 만들지”를 Architecture/SDS로 확정한다.

동시에, 개발 Agent를 활용하여 Main Navigation 화면을 빠르게 구현해 Early Feedback(가독/흐름/데모 가능성)을 확보한다.
단, 구현은 EA 산출물의 의사결정(설계)과 충돌하지 않도록 SRS/Architecture 범위 내에서 진행한다.

## 목적

1. 전시 일정(5월) 역산 기준으로, 3월 내에 기구·S/W 설계 기준선을 확보해 4월 제작/통합 리스크를 낮춘다.
2. “문서와 데이터 기반” Gate 원칙에 맞게 설계 의사결정 기록을 남기고, 4월 구현/통합 단계로 매끄럽게 연결한다.
3. Agent 기반 구현을 통해 UI 흐름을 조기에 가시화하고, 주요 리스크를 선제적으로 노출한다.

## 수요자 요구 사항

- 5월 중 전시 가능한 시작품(데모 시스템)
- 연말까지 인증 투입 가능한 시제품 로드맵과 정합
- 개발은 Gate 기반으로 진행되어 범위·의사결정·변경관리가 명확할 것

## 현재 상태

### As-Is

- PA 착수(2월)로 Scope/마일스톤/운영체계 합의 진행
- Navigation View Application 존재 및 오차 측정 체계 초안 존재(1월)

### To-Be

- 5월 SIDEX 데모 시스템의 기구 설계·S/W 설계 기준선이 확보되고,
- EA 산출물(SRS/Architecture/SDS)이 정리되며,
- Main Navigation 화면 구현 결과물(프로토타입/초기 빌드)이 확보된 상태

## 기대하는 결과물

1. **EA 산출물 세트 v1.0:** SRS v1.0 / Architecture v1.0 / SDS v0.8(초안 수준 가능)
2. **Demo System 기구 디자인 패키지 v1.0:** 레이아웃/기구 도면(또는 CAD), 조립 개념, BoM 초안
3. **Demo System S/W 디자인 패키지 v1.0:** 화면 흐름(IA/Flow), 상태/이벤트 정의, 주요 인터랙션 스펙
4. **Main Navigation 화면 구현 결과물 v0.5:** Agent 기반 구현(빌드/데모 영상/코드) + 구현 로그
5. **결정 로그(Decision Log) v1.0:** EA 설계 결정/보류/TBD 목록

## Performance Objectives

### 성과 목표 1: Demo System 기구 디자인 패키지 v1.0

**고정변수목표:** 기구 설계 자료 v1.0(레이아웃·CAD/도면·조립 컨셉·BoM 초안)

- **공략 전략:** SIDEX 데모 시나리오 기준 필수 구성만 고정(이동/고정/거치/운용 동선), 제조/조달 리드타임 반영해 “제작 가능한 설계”로 정리

**변동변수목표(센터장):** 기구 Scope 확정(필수/옵션 분리)

- **공략 전략:** 5월 데모에 필요한 최소 구성(필수)과 있으면 좋은 구성(옵션)을 분리하고, Gate Keeping 기준으로 범위 통제

### 성과 목표 2: Demo System S/W 디자인 패키지 v1.0

**고정변수목표:** S/W 디자인 패키지 v1.0(화면 Flow·상태/이벤트·인터랙션 스펙)

- **공략 전략:** “데모 성공” 기준의 최소 화면/기능만 확정(메인 내비 화면 중심), 화면별 입력/출력/오류 상태 정의

**변동변수목표(센터장):** SIDEX 데모 시나리오 v1.0(사용자 여정)

- **공략 전략:** 데모 진행 순서(시작→정합/표시→오차 정보→종료)를 1페이지로 고정하고, 시나리오 변경은 Decision Log로 관리

### 성과 목표 3: EA 산출물 세트 v1.0 (SRS/Architecture/SDS)

**고정변수목표:** SRS v1.0 + Architecture v1.0

- **공략 전략:** 요구사항(Design Input)을 데모 범위로 완결시키고, 아키텍처(모듈/데이터 흐름/인터페이스/로그)를 확정하여 EA 목표 달성

**변동변수목표(센터장):** SDS v0.8(구현 직전 수준의 설계 초안)

- **공략 전략:** 구현 리스크가 큰 모듈(내비 뷰/오차 표시/로그·리플레이) 중심으로 상세 설계를 먼저 고정하고, 나머지는 TBD를 명확히 남김

### 성과 목표 4: Main Navigation 화면 구현 결과물 v0.5 (Agent 활용)

**고정변수목표:** Main Navigation 화면 빌드 v0.5

- **공략 전략:** Agent에게 역할을 명확히 분리(레이아웃/상태/데이터 바인딩/모킹), “빌드 가능한 결과물”을 매주 확보

**변동변수목표(센터장):** Agent 개발 운영 규칙 v1.0(품질/보안/형상관리)

- **공략 전략:** CMP(형상관리) 원칙과 정합되게 PR 규칙/코드리뷰/산출물 기록 방식 정의(Agent 산출물도 Decision Log에 연결)
