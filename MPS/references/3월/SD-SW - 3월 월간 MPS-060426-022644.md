# SW - 3월 월간 MPS

> **SW-2026년 3월 월간 MPS**

## Mission

Smile Design 2.0 PA 단계 실행 및 기본 기능 Agile 개발을 통한 Gate 기반 개발 시스템 검증

## 개요

2026년 3월은 2월 Kick-off에서 합의된 과제 목적/기간/Scope를 기준선(Baseline)으로 삼아 Gate 기반 개발 프로세스의 PA(Project Approval) 단계를 본격 실행한다.

동시에, 과제 리스크를 조기에 드러내고 실제 개발 흐름의 타당성을 확인하기 위해 핵심 기본 기능을 Agile하게 개발하며, 이 과정을 통해 Gate 기반 개발 시스템(산출물/승인/검토 체계) 자체도 함께 검증한다.

## 목적

1. Kick-off 합의 내용을 기반으로 PA 단계 필수 산출물을 완성/정교화하여 착수 승인 체계를 안정화
2. 기본 기능 개발을 병행하여 기술 리스크를 조기에 식별하고, 과제 진행 가능성을 실제로 검증
3. Gate 기반 개발 시스템(문서/데이터 기반 승인, 검토 절차, 역할/승인 체계)을 운영해보며 실효성 검증

## 수요자 요구 사항

- Scope/일정/역할이 명확한 상태에서 통제 가능한 개발 착수
- 초기 단계부터 리스크/형상/보안 관점이 포함된 개발 거버넌스 운영
- 빠른 기능 확인(초기 실행물)을 통한 방향성 확신과 리스크 조기 제거

## 현재 상태

### As-Is

- Kick-off에서 과제 목적/기간/Scope에 대한 합의가 존재하나
- PA 단계 산출물의 완성도 및 운영 방식은 실제 적용/검증이 필요

### To-Be

- PA 단계 필수 산출물 세트가 기준선으로 확정되고, 승인/변경관리 체계가 동작
- 기본 기능의 초기 실행물을 확보하여 기술 리스크를 조기에 가시화
- Gate 기반 개발 시스템 운영 결과(개선점 포함)가 검증 리포트로 정리됨

## 기대하는 결과물

1. **PA 단계 산출물 세트**(확정본/초안 포함): SDP, RMP, CMP, 보안 및 유지보수 계획, RA 전략/제품 등급(Class) 정리
2. **기본 기능 초기 실행물**(Prototype/Alpha)
3. **Gate 기반 개발 시스템 운영 검증 리포트**(개선안 포함)

## Performance Objectives

### 성과 목표 1

**목표:** Smile Design 2.0 PA 단계 산출물 세트(운영 가능한 기준선)

**개요:**
PA 단계에서 요구되는 필수 산출물을 실제 개발 운영에 사용할 수 있는 수준으로 완성하고, 승인/변경/형상/보안 체계가 실제로 동작하도록 기준선을 확정한다.

#### 고정변수목표 1: PA 산출물 세트

- SDP(개발 계획서)
- RMP(위험관리계획서)
- CMP(형상관리계획서)
- 보안 및 유지보수 계획서
- RA 전략 및 제품 등급(Class) 정리

**공략 전략**

1. Kick-off 합의사항을 문서 기준선으로 반영하고 변경관리 규칙(변경 요청/승인/기록) 정의
2. QA 참여 기준과 산출물 버전 관리 규칙을 명확히 하여 Gate 운영 기반 확보
3. PA Gate 운영을 “문서와 데이터 기반”으로 수행할 수 있도록 증적(근거) 구조화

### 성과 목표 2

**목표:** Smile Design 2.0 기본 기능 초기 실행물(Prototype/Alpha)

**개요:**
EA(요구사항/아키텍처 확정)로 넘어가기 전에, 기술 리스크가 큰 구간을 조기에 확인할 수 있도록 기본 기능을 Agile하게 구현하여 초기 실행물을 확보한다.

#### 고정변수목표 2: 기본 기능 Prototype/Alpha 실행물

**공략 전략**

1. 최소 단위 실행 흐름(MVP Path) 중심으로 구현 범위를 제한
2. “데이터 입력 → 핵심 처리 → 결과 확인”이 가능한 최소 루프 확보
3. 구현 과정에서 발생한 리스크/제약 사항을 문서화하여 PA 리스크 관리에 반영

### 성과 목표 3

**목표:** Gate 기반 개발 시스템 운영 검증 리포트(개선안 포함)

**개요:**
PA 단계 산출물 작성/승인/변경관리와 Agile 개발을 병행 운영하면서, Gate 기반 개발 시스템이 실제 조직/일정/개발 흐름에 적합한지 검증하고 개선안을 도출한다.

#### 고정변수목표 3

Gate 운영 검증 리포트(운영 결과, 문제점, 개선안, 표준 템플릿/체크리스트)

#### 변동변수목표 1

Stakeholder 승인/검토 리드타임(문서 승인 소요) 단축 또는 안정화 결과 (예: 승인 지연 원인 분석 및 개선 액션 도출)

**공략 전략**

1. Gate 산출물 템플릿/체크리스트를 실제 업무 흐름에 적용해보고 개선점 수집
2. Go/Kill/Hold/Recycle 의사결정이 가능한 수준의 근거/증적 구조가 작동하는지 점검
3. 개발팀/QA/Stakeholder 관점의 병목(승인, 리뷰, 변경관리) 항목을 도출하고 운영 규칙 보정

| 리스크 | 대응방안 |

| 리스크 | 대응방안 |
| --- | --- |
| PA 문서 작업과 개발 병행으로 리소스 분산 | “필수 산출물 우선” 원칙으로 범위 통제, 기본 기능은 MVP Path 중심으로 제한 |
| Gate 프로세스가 과도하게 무거워질 위험 | 운영 검증 리포트에서 템플릿 단순화/승인 단계 최적화 개선안 도출 |
