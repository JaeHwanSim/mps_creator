# AI - '26 3월 월간 MPS

> **2026년 3월 월간 MPS (R&D 센터 AI 부분)**

**과제:** JIRA 기반 Gate Process 관리 시스템 적용·견고화 + Goose 기반 AI Agent 개발 시스템 적용·견고화

---

## Mission

2월까지 구축된 JIRA 기반 Gate Process 관리 시스템(미완성)과 Goose 기반 AI Agent 개발 시스템(미완성)을 실제 개발 업무에 즉시 적용하면서, 운영 안정성(견고화)과 추적성(Traceability)을 확보한다.

> **원칙:** 완벽을 기다리지 않고 “적용 → 피드백 → 개선” 사이클로 완성도를 끌어올린다.

## 개요

R&D Gate 기반 개발 프로세스는 단계별 불확실성을 통제하고 규제 준수성을 동시에 확보하기 위해, 각 Gate를 문서와 데이터 기반 **Go/Kill/Hold/Recycle**로 운영한다.

3월은 2월에 만든 시스템들을 “완성”하기보다,

1. JIRA로 Gate 산출물/승인/결정 로그를 관리하고
2. Goose 기반 AI Agent 개발 시스템으로 개발자 협업(이슈→코드→테스트→리뷰→문서화)을 실행

하면서, 실사용에서 드러나는 병목/결함/운영 리스크를 빠르게 제거해 견고한 기준선(Baseline)을 만드는 달이다.

## 목적

1. JIRA에서 Gate(Process) 운영이 실제로 돌아가도록 적용하고, Gate별 추적성(산출물·승인·결정)을 확보한다.
2. Goose 기반 AI Agent 개발 시스템을 실제 개발에 적용하여 개발 협업 리드타임을 단축하고 품질 게이트를 강화한다.
3. “개발 완료와 동시에 의료기기 인증(인허가) 투입이 가능한 준비 상태”를 만들기 위한 최소 요건을 3월 안에 고정한다.

## 수요자 요구 사항

개발이 완료됨과 동시에 의료기기 인증(인허가)을 투입할 수 있도록 준비할 것.

## 현재 상태

### As-Is

- JIRA 기반 Gate Process 관리 시스템이 미완성 상태로, Gate 산출물/승인/결정 로그가 일관되게 축적되지 않음.
- Goose 기반 AI Agent 개발 시스템은 구성되어 있으나, 실제 개발 사이클에서 반복 적용 경험과 운영 규칙이 부족함.
- Gate 프로세스는 “문서와 데이터” 중심 운영이 원칙이나, 실사용 연결(이슈–문서–코드)이 아직 느슨함.

### To-Be

- JIRA에서 Gate별 필수 산출물 링크, 승인 상태, Decision Log가 누락 없이 관리되어 “Gate 운영이 시스템으로 굴러감”.
- Goose 기반 AI Agent가 개발 흐름(이슈→코드→테스트→PR→문서)에서 반복 사용되며, 품질/보안 기준을 자동으로 보조함.
- 산출물·이슈·코드가 연결되어 DHF(Design History File) 관점의 추적성이 확보되고, EA/ER 단계 진입이 빨라짐.

## 기대하는 결과물

1. **JIRA 기반 Gate Process 운영 기준선 v1.0** (워크플로우/이슈타입/필수필드/승인 규칙/Decision Log 템플릿)
2. **Gate 운영 대시보드 v1.0** (Gate별 진행률, 승인 대기, 산출물 누락, 리스크 현황)
3. **Goose 기반 AI Agent 개발 시스템 운영 기준선 v0.5** (Agent 역할/권한/프롬프트 패키지/운영 가이드)
4. **Agent 적용 개발 실적 리포트 v1.0** (PR 4건 이상, 테스트/리뷰/문서화 자동화 적용 결과, 개선 백로그)
5. **규제(인허가) 투입 준비 체크리스트 v1.0** (Gate 산출물/추적성/보안/형상/위험관리 관점 최소요건)

## Performance Objectives

### 성과 목표 1

**목표:** JIRA 기반 Gate Process 운영 기준선 v1.0 + Gate 운영 대시보드 v1.0

**개요:**
Gate 기반 개발 프로세스의 “문서와 데이터 기반 의사결정”을 JIRA에서 실제로 굴릴 수 있도록, Gate 운영의 최소 기준선(Workflow/정책/필드/승인/결정 로그)을 확정하고 운영 대시보드를 구축한다.

#### 고정변수목표 1: JIRA Gate 워크플로우/이슈 체계 v1.0

**공략 전략**

1. Gate 단위 이슈 타입 정의(PA/EA/ER 등) 및 필수 필드(산출물 링크, 승인자, Due date, 상태) 표준화
2. Gate별 “필수 산출물 체크”를 이슈 필드/체크리스트로 고정하여 누락을 시스템적으로 차단
3. Decision Log 템플릿을 JIRA에 내재화(결정/보류/추가확인/담당/기한)하여 후속 지연 통제

#### 고정변수목표 2: Gate 운영 대시보드 v1.0

**공략 전략**

1. Gate 진행률/승인 대기/산출물 누락/리스크 Top N을 한 화면에서 확인하도록 구성
2. 주간 운영 리뷰에서 “대시보드 기반 의사결정”이 되도록 운영 루틴(주 1회) 고정

#### 변동변수목표: Pilot 운영 리포트 v1.0 (JIRA 기반 Gate 운영)

**공략 전략**

1. 실제 개발 업무(최소 1개 기능/이슈 묶음)에 Gate 운영을 적용해 누락/병목/저항 포인트를 수집
2. 운영 리포트에 개선 우선순위를 포함하고, 4월 개선 백로그로 전환하여 통제

### 성과 목표 2

**목표:** Goose 기반 AI Agent 개발 시스템 운영 기준선 v0.5 + Agent 적용 개발 실적 리포트 v1.0

**개요:**
Goose 기반 AI Agent 시스템을 “개발자가 매일 쓰는 도구”로 만들기 위해, 실제 개발에 즉시 적용하면서 품질/보안/리뷰 규칙을 포함한 운영 기준선을 확정한다.

> 인허가 투입 준비를 위해, 코드/문서/테스트의 형상·추적성이 함께 강화되어야 함

#### 고정변수목표 1: Goose Agent 역할/권한/운영 가이드 v0.5

**공략 전략**

1. Agent 역할 정의: 이슈 정리, 코드 스캐폴딩, 테스트 생성, PR 리뷰, 릴리즈 노트/문서화
2. 권한/금지 규칙 고정: “Agent 제안=초안, 사람 승인=최종” 및 민감정보 처리(마스킹/금지 프롬프트) 명문화
3. 개발자 협업 규칙(리뷰 체크리스트/DoD/테스트 필수)을 운영 가이드에 포함

#### 고정변수목표 2: Repo/CI/PR 표준 + 품질 게이트 v0.5

**공략 전략**

1. PR 템플릿/리뷰 체크리스트/테스트 실행 규칙을 표준으로 고정
2. CI 품질 게이트(필수 테스트/정적 분석/필수 체크) 미통과 시 병합 불가로 강제

#### 변동변수목표: Agent 적용 개발 실적 리포트 v1.0 (PR 4건 이상 + 개선 백로그)

**공략 전략**

1. 실제 개발 작업에서 PR 4건 이상에 Agent 사용을 적용(최소 2명 개발자 참여)
2. 리포트에 “시간 절감/결함 감소/리뷰 리드타임 변화/문서 생성 품질”을 정리
3. 발견된 문제를 개선 백로그로 전환하여 4월 목표 입력으로 사용
