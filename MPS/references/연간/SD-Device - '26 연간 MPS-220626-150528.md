# Device - '26 연간 MPS (센터 MPS — Robot · Dynamic CAIS)

> **2026년 연간 MPS Planning (R&D 센터 Device)**
> **핵심:** 단일 SW로 Human-Glass / Robot 두 케이스 지원, AR은 H2 고도화
>
> *센터 MPS(입력 자료). 내 MPS(`MPS/created/`) 도출 시 참고.*

## Mission

- **H1:** 전시회 출품 가능한 MVP 통합 데모 시스템 완성 (단일 SW에서 Human-Glass 모드·Robot-Assist 모드 지원)
- **H2:** Dynamic CAIS 통합 프로토타입 v1.0 완성 및 인허가(RA) 투입 패키지 준비

## 개요

2026년은 **구현–통합–검증–전시–인허가 준비** 단계다.
단일 애플리케이션에서 **Human-Glass**(의사 착용 안경으로 안내 시야 제공)와 **Robot-Assist**(로봇 보조/제어) 두 케이스를 런타임 프로파일로 운용하는 구조를 확립한다.
H1은 Monitor Glasses 중심의 MVP로 전시 출품, H2는 AR Glasses 통합과 안전성·정확도 고도화 및 RA 착수로 구성한다.

## 목적

1. 전시 데모 성공(H1) 및 이중 모드 단일 SW 전략 검증
2. 예외·복구·안전성 중심의 성능 고도화(H2)와 규제 준비도 확보
3. 2027 상용화 로드맵 정합 유지

## 수요자 요구 사항

- Human-Glass / Robot-Assist 두 케이스의 단일 SW 지원
- Monitor Glasses(H1) / AR Glasses(H2) 단계적 대응
- 변경 관리와 형상/문서 추적성 유지, 전시(H1)/RA 착수(H2) 일정 준수

## 현재 상태

### As-Is ('25 말)
- Direct Planning SRS/HLD v1, 글래스 통합 요구·설계 정의
- 특이점 대응 설계/검증 계획 보유
- 모바일 카트 명세·BoM 및 일부 PoC 결과 보유

### To-Be ('26 말)
- Dynamic CAIS 통합 프로토타입 v1.0 (단일 SW 이중 모드·AR 통합 포함)
- 전시회용 MVP 데모 번들 (스크립트·체크리스트·리플레이 로그)
- 성능/안전 예비 검증 리포트 (정확도·지연·복구·폴백)
- Monitor/AR Glasses 적용 가이드
- 인허가 투입 패키지 초안 (추적성 매트릭스, RMF 초안, IFU 개요)

## 기대하는 결과물

## Performance Objectives

> 성과 목표 수를 축소하고, 각 목표는 결과물(명사형)로 제시한다.

### 성과 목표 1: MVP 통합 데모 시스템 (H1 전시 출품)

**고정변수목표:** 전시 데모 번들 (표준 시연 스크립트·체크리스트·리플레이 로그)
- 공략전략: 표준 케이스 기준선 확정 → 리허설 반복 → 결함/개선 백로그 운영

**변동변수목표:** 가독성·지연 벤치 리포트 (Monitor Glasses 중심)
- 공략전략: 권장 표시 프로파일 기반 측정 → 합격 기준 정의 → 개선안 반영

**Risk**
- **일정:** 통합 지연/전시 충돌 → H1 범위 최소화·게이트 리뷰, H2 고도화 분리

### 성과 목표 2: Dual-Mode 통합 애플리케이션 v1 (단일 SW Human-Glass / Robot-Assist)

**고정변수목표:** 런타임 프로파일 사양서 (모드 관리·공통 코어/모드 어댑터·폴백 정책)
- 공략전략: 공통 모듈(데이터·가이드·로그)과 모드별 어댑터(Glass 표출·Robot 제어) 분리 설계 → 인터페이스(IDL) 고정

**변동변수목표:** 리스크 매트릭스 (자원 충돌·모드 전환·폴백)
- 공략전략: 충돌 자원 분리/락 정책·전환 시나리오·Failover 절차 정의 → 벤치 검증 계획 수립

**Risk**
- **내부(복잡도):** 단일 SW 이중 모드 유지보수 → 공통/전용 모듈 분리, 인터페이스 고정, 회귀 자동화

### 성과 목표 3: Dynamic CAIS 통합 프로토타입 v1.0 (H2)

**고정변수목표:** AR Glasses 통합 패키지 v1 (표시 항목·세션/권한·폴백)
- 공략전략: 벤더 가이드/샘플 매핑 → 시나리오 오버레이 설계 → 복구/알림 흐름 정립

**변동변수목표:** 성능/안전 예비 검증 리포트 v1 (정확도·지연·복구)
- 공략전략: 벤치·모의 임상 데이터 수집 → 합격 기준 대비 평가 → 튜닝 가이드 업데이트

**Risk**
- **외부(장비):** 글래스/트래킹/엔진 변경 → 대체 목록·버전 고정·CR 관리
- **내부(성능):** 지연/정확도 기준 미충족 → 프로파일 튜닝·표시 축약·폴백 강화

### 성과 목표 4: 인허가(RA) 투입 패키지 초안 (H2)

**고정변수목표:** 추적성 매트릭스(요구–설계–시험 링크) & 위험관리 파일(RMF) 초안
- 공략전략: 변경요청(CR) 체계 운영 → 분기 문서 심사 → 형상/버전 관리 정례화

**변동변수목표:** IFU 개요·시험 계획서 초안
- 공략전략: 표준/가이드 맵핑(MDR/KFDA 등) → 성능 기준/샘플 데이터 정의 → 계획서 초안 확정

**Risk**
- **내부(규제 문서화):** 초기 누락 → H1부터 추적성/형상 관리 일상화, 분기 문서 심사
