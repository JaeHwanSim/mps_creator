# SW - '26 4월 월간 MPS (내 MPS)

> **2026년 4월 내 MPS — CBCT Segmentation·VD Landmark, 재현 가능한 파이프라인 코드 기준선 v0.7 확보 (이후 지속 개선)**
>
> **입력 자료:** `MPS/references/4월/SD-SW` 센터 MPS 참고

## Mission

3월 Prototype/Alpha·Gate(PA/CMP) 기반 위에서 CBCT Segmentation·VD Landmark 두 파이프라인의 **재현 가능한 코드 기준선 v0.7**을 확보하고, 5월 고도화·SAD 2.0 통합으로 이어지는 **개선 루프**를 고정한다.

## 개요

3월은 개발·형상(PA/CMP) 기반을 다졌다. 4월은 그 위에서 두 파이프라인을 "작동하는 첫 기준선(v0.7)"으로 다진다. AI팀 모델은 그대로 두고, 그 모델을 감싸는 엔지니어링 래핑(입출력 스펙·평가 harness·QC 자동화)을 담당한다. 핵심은 **"평가/QC → 보정 → 재평가"** 개선 루프를 고정하는 것이다.

- **루프 흐름(text):** 성과 목표 1(Segmentation 파이프라인)에서 구축한 **공통 엔지니어링 기반**(표준 입출력 스펙·데이터 로더·추론 wrapper·평가 harness·QC 자동화) → 성과 목표 2(VD Landmark 파이프라인)가 재사용·확장 → 두 파이프라인의 평가 harness·QC 결과가 데이터/라벨 기준 보정으로 환류 → 재평가(지속 개선)

## 목적

1. CBCT Segmentation·VD Landmark의 **재현 가능한 파이프라인 코드 기준선 v0.7**을 확보한다 (표준 입출력 스펙·config·시드·의존성 고정).
2. 공통 엔지니어링 기반(표준 입출력 스펙·자동 평가 harness·QC 자동화) v0.7을 정립해 이후 고도화·통합의 엔지니어링 토대를 마련한다.
3. 평가 harness·QC 결과가 데이터/기준 보정으로 이어지는 **지속 개선 루프**를 가동한다.

## 수요자 요구 사항

- **연구·AI팀:** 모델 v0.x를 감싸는 재현 가능한 추론 파이프라인 + 표준 입출력 스펙 + 자동 평가 harness
- **제품·SAD 2.0 통합팀:** 표준화된 입출력 인터페이스(CBCT → mask/landmark)와 샘플 E2E 결과
- **데이터·QC:** 라벨 포맷 스펙과 QC 자동 체크 기준

## 현재 상태

### As-Is

- 3월 Prototype/Alpha 수준의 실험 코드가 산발적으로 존재, 라벨/정답셋 부족
- 입출력 스펙·재현성(시드/config/의존성 고정) 미비, 자동 평가 harness·QC 자동화 부재
- PA/CMP 형상 체계는 기준선 확보, 단 파이프라인 코드에 형상 규칙 적용은 미진행

### To-Be

- 두 파이프라인 재현 가능 코드 기준선 v0.7 + 표준 입출력 스펙 + 자동 평가 harness + QC 자동화 + 샘플 E2E
- 형상(Git tag·config lock) 적용, 평가/QC 환류 루프 가동

## 기대하는 결과물

3. **공통 엔지니어링 기반 v0.7** (표준 입출력 스펙·데이터 로더·평가 harness·QC 자동화·재현성 가이드)

## Performance Objectives

### 성과 목표 1: CBCT 하악·치아 Segmentation 파이프라인 (코드 기준선 v0.7)

> 입력 CBCT로부터 하악/치아 마스크를 산출하는 재현 가능한 추론 파이프라인과 평가 harness를 확보한다. 모델은 AI팀 v0.x를 그대로 사용하고, 그 모델을 감싸는 엔지니어링 래핑을 담당한다. 이 PO에서 구축한 공통 엔지니어링 기반은 성과 목표 2가 재사용한다.

**고정변수목표**
- 재현 가능 추론 파이프라인 v0.7: 표준 입출력 스펙 정의(NIfTI/PNG/JSON), config + 시드 + 의존성 고정(requirements/Docker), CLI/함수 인터페이스, E2E 스크립트 + 회귀용 샘플 케이스
  - 공략전략: MVP 경로(하악·치아 핵심 구조)를 먼저 표준화, 적은 케이스라도 E2E 가동을 우선 확보, 입출력 스펙을 코드와 문서로 이중 고정
- 자동 평가 harness v0.7: IoU/DSC 메트릭 계산 + 시각화 자동화 + QC 통과/실패 자동 체크
  - 공략전략: 메트릭 계산·시각화를 스크립트로 자동화, QC 기준(FOV·분포 태깅)을 코드화, 실패/경계 케이스는 harness가 자동 기록

**변동변수목표**
- 성능 리포트 자동 생성 v0.7: 수치 + 샘플 시각화 + 경계 케이스 성능 저하 요인 자동 추출
  - 공략전략: 평가 harness 결과를 리포트 템플릿으로 자동 변환, 데이터 품질/분포 리스크를 가정하고 주간 점검, 라벨 기준 불일치 시 라벨 가이드→QC 코드 우선 보정

### 성과 목표 2: VD(고경) Landmark Detection 파이프라인 (코드 기준선 v0.7) 및 검증

> VD 측정용 최소 Landmark 세트를 정의하고(구개평면 ANS–PNS을 primary 기준), 검출 모델을 감싸는 재현 가능 추론 파이프라인과 검증 harness를 확보한다. 성과 목표 1의 공통 엔지니어링 기반을 재사용·확장한다.

**고정변수목표**
- VD Landmark 추론 파이프라인 v0.7: 최소 Landmark 세트(ANS/PNS/ROr/LOr/RMeF/LMeF 중심), 입출력 스펙(JSON), config·시드 고정, E2E 스크립트
  - 공략전략: 성과 목표 1의 입출력 스펙·데이터 로더·평가 harness를 재사용, 구개평면 primary 원칙을 코드에 고정, 수동 정답 소수 케이스로 자동 결과 비교 회귀
- VD 관점 검증 harness v0.7: OP 후보 산출 근거 포함 + CBCT overlay 보조 검증 스크립트
  - 공략전략: QC 통과 케이스 우선 사용, 오검출 패턴을 harness가 자동 수집→피드백, VD에 직접 연결되는 지표 중심으로 간결한 리포트 자동화

**변동변수목표**
- Landmark 검출 정밀/재현 평가 자동화 + OP 기울기·높이 후보 생성의 임상 납득성 검증(샘플 근거) v0.7
  - 공략전략: 평가 harness로 정밀/재현 산출, 후보 타당성은 overlay 스크립트로 보조 검증, 결과를 공통 QC로 환류

## Risks

> 미션 전체 관점. 외부환경 1개 이상 + 내부역량 1개 이상.

- **외부(데이터 수급·분포):** 라벨/데이터 수급 지연 + CBCT 분포 편차(Full Face vs 하악 FOV) → 필수 케이스 우선·범위 고정, 분포 태깅 + QC/리포트 분리 코드화
- **외부(인터페이스 의존):** AI팀 v0.x 모델 인터페이스 변경 시 파이프라인 재작업 → 입출력 스펙을 안정 계약으로 고정·버전 태깅, 모델 변경은 adapter 레이어에서 흡수
- **내부(재현성·형상):** 시드/config/의존성 미고정으로 결과 변동 → config lock + Git tag + Docker/requirements 고정, 3월 CMP 형상 규칙 적용
- **내부(QC·기준):** QC 기준·라벨 기준 미비 → 라벨·QC 가이드 + QC 자동화 스크립트 우선 확정, 예외는 설정(변수)으로 관리
